# 第一次提问

这页只解决一件事：

**如果你第一次打开 Codex，不知道该问什么，就直接复制这里的提示词 prompt。**

新手最重要的不是问得多，而是先问对。

---

## 第一条原则：任务要小

第一次不要一上来问：

```text
帮我做一个网站。
帮我做一个系统。
帮我完成一个项目。
```

这些问题太大，容易让你更乱。

第一次更适合问：

```text
请把我当成完全新手，帮我完成一个很小的任务。
```

先小，再大。先成功一次，再学更多。

---

## 第二条原则：告诉 Codex 你的水平

如果你是小白，就直接说：

```text
请把我当成完全新手。
不要默认我懂代码。
不要一下子讲太多专业词。
```

这不是丢人。这是在给 Codex 正确的上下文。

---

## 如果你只复制一条，就用这条

```text
请把我当成完全新手。
不要默认我懂代码，也不要讲太多专业词。

请你：
1. 用最简单的话解释 Codex 能帮我做什么
2. 告诉我现在第一步该做什么
3. 告诉我哪些内容我现在可以先跳过
4. 给我一个 5 分钟内能完成的小任务
```

这条 prompt 的目标不是学完 Codex。它的目标是帮你开始。

---

## Prompt 1：先搞懂 Codex 是什么

```text
请把我当成完全新手。
不要默认我懂代码，也不要讲太多专业词。

请你用最简单的话告诉我：
1. Codex 是什么
2. 它和普通聊天 AI 有什么不一样
3. 它最适合帮我做什么
4. 我第一次使用时最容易误解什么
```

适合在你完全没概念时使用。

---

## Prompt 2：让我知道第一步该做什么

```text
我现在是完全新手。
请不要一下子讲太多内容。

请你只告诉我：
1. 我现在第一步该做什么
2. 第二步该做什么
3. 哪些内容我现在先不用学
4. 如果我卡住了，应该先检查什么
```

适合在你看了很多东西但不知道从哪开始时使用。

---

## Prompt 3：给我一个第一次小任务

```text
请把我当成完全新手。
我想先试一个最简单的小任务，感受一下 Codex 能怎么帮我。

请你给我：
1. 3 个适合新手马上尝试的小任务
2. 每个任务为什么适合新手
3. 你建议我先选哪一个
4. 完成这个任务的判断标准
```

适合第一次真正使用 Codex。

---

## Prompt 4：帮我整理一个模糊想法

```text
我现在脑子里有一个模糊想法，但我说不太清楚。
请你把我当成完全新手，帮我把这个想法整理清楚。

请你帮我：
1. 先用一句话复述你理解的想法
2. 把这个想法整理成更清楚的描述
3. 拆成 3 个简单步骤
4. 告诉我最适合先做哪一步
```

适合你有想法，但不知道怎么表达时使用。

---

## Prompt 5：解释一段看不懂的话

```text
请把我当成完全新手。
下面这段话我看不懂，请你不要直接使用专业术语解释。

请你：
1. 用小白能懂的话解释
2. 举一个生活化例子
3. 告诉我这段话和 Codex 使用有什么关系
4. 告诉我现在需不需要深入学习它

内容如下：
【把你看不懂的内容贴在这里】
```

适合看官方文档、教程或报错时使用。

---

## Prompt 6：解释报错并给下一步

```text
请把我当成完全新手。
我遇到了下面这个报错，请你先不要急着让我重装所有东西。

请你：
1. 用简单的话解释这个报错可能是什么意思
2. 列出最可能的 3 个原因
3. 告诉我应该先检查哪一步
4. 如果需要运行命令，请说明这个命令只是检查还是会修改系统

报错如下：
【把报错原文贴在这里】
```

适合 Windows、PowerShell、npm、Git 或 Codex 启动失败时使用。

---

## Prompt 7：让 Codex 先确认再行动

如果你准备让 Codex 修改文件或运行命令，先用这个：

```text
在你动手之前，请先确认你理解的任务。

请你先回答：
1. 你认为我要做什么
2. 你准备改哪些文件或运行哪些命令
3. 哪些操作只是读取或检查
4. 哪些操作会修改文件或系统
5. 我是否需要先备份文件夹，或在 Git 里做一次 checkpoint
6. 有没有风险或不确定的地方

在我确认之前，不要修改文件。
```

这条很重要。新手阶段不要把所有操作都自动交出去。如果你已经会 Git，改文件前后可以各做一次提交；如果不会 Git，至少先复制一份文件夹作为备份。

---

## 好 prompt 的小公式

你可以按这个顺序写：

```text
请把我当成【你的水平】。
我现在想要【很小的目标】。
请你不要【你不想要的方式】。
请你按【明确格式】回答。
```

例子：

```text
请把我当成完全新手。
我现在想要先完成一次 Codex 小任务。
请你不要讲太多术语，也不要让我做大项目。
请你按 1、2、3 步告诉我下一步。
```

---

## 第一次提问完成标准

你完成下面任意一项，就算成功：

- Codex 用你能懂的话解释了一个概念。
- Codex 帮你选了一个小任务。
- Codex 帮你把一个想法拆成步骤。
- Codex 帮你解释了一个报错。
- 你知道下一步该做什么。

这就是第一次小成功。
