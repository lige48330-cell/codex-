# 从这里开始

欢迎来到 Codex 小白入门课。

这套课不是让你第一天学完所有概念。今天只做一件事：

**先让 Codex 帮你完成一次很小、很明确的成功。**

只要这一步走通，后面的概念才会变得有意义。

---

## 最短入口

如果你现在只想知道“第一步到底是什么”，那就按这 4 步：

1. 下载整个仓库 ZIP
2. 完整解压
3. 双击 `start-here.cmd`
4. 如果卡住，立刻去 [排错中心](./05-troubleshooting.md)

如果你连 CMD 都没用过，不要硬扛，直接跳去 [命令行小白课](./04-cmd-for-beginners.md)。

---

## 这页适合谁

如果你有下面任意一种情况，就从这里开始：

- 我完全不知道 Codex 是什么。
- 我不会写代码，但想试试看 Codex 能不能帮我。
- 我装了东西，但不知道下一步做什么。
- 我看到了很多术语，比如 CLI、IDE、Agent、MCP，有点晕。
- 我不想先啃一大堆文档，只想先用起来。

你不需要先成为程序员。你只需要先完成一个小动作。

---

## 如果你刚从 GitHub 下载

先确认你已经做完这几步：

1. 点击 GitHub 页面里的 `Code`。
2. 选择 `Download ZIP`。
3. 把 ZIP 文件完整解压。
4. 打开解压后的文件夹。

ZIP 文件名可能不是 `codex-github-release.zip`，这很正常。重点不是压缩包名字，而是解压后的文件夹里有没有下面这些内容。

这个文件夹里应该能看到：

```text
README.md
docs
start-here.cmd
check-codex-env.ps1
```

如果你还没看到这些文件，先不要运行命令。回到解压后的文件夹，再继续往下看。

---

## 今天的目标

读完前 4 页后，你应该做到：

1. 知道 Codex 大概是干什么的。
2. 知道自己第一步该做什么。
3. 完成一次环境检查或启动尝试。
4. 复制一条提示词 prompt，让 Codex 帮你完成一个小任务。

这就够了。

第一次成功不需要是“做完一个项目”。第一次成功可以很小，比如：

- 让 Codex 解释一段你看不懂的话。
- 让 Codex 总结一段文字。
- 让 Codex 把你的模糊想法整理成步骤。
- 让 Codex 解释一个报错，并告诉你下一步该查什么。

## 30 分钟路线

如果你不知道从哪开始，按这个顺序走：

1. 读本页，先知道今天只做什么。
2. 读 [Codex 是什么](./01-what-is-codex.md)，先建立基本理解。
3. 读 [第一次运行](./02-first-run.md)，完成环境检查或第一次启动。
4. 如果你卡在 CMD / PowerShell，再临时跳去 [命令行小白课](./04-cmd-for-beginners.md)，补完回到第 3 步。
5. 读 [第一次提问](./03-first-prompts.md)，复制一条提示词直接试。
6. 如果卡住了，去 [排错中心](./05-troubleshooting.md)。

命令行课是分支页，不是第一天必须完整学完的主线。你只需要补到能进入课程文件夹、能运行检查脚本就可以回来。

不要一上来翻完所有文件。小白最容易卡住的地方，不是学得太少，而是第一天学得太散。

## 今天先不要学什么

这些东西以后会有用，但不适合第一天硬学：

- MCP
- Skill
- Subagent
- 复杂配置
- 大型项目自动化
- 权限、安全沙箱、深度自定义

如果你现在完全是新手，先把它们当成“以后再说”。没听过这些词，也不用现在去查。今天只关心：

**Codex 能不能帮我完成一个小任务？**

---

## Windows 用户先看这里

本仓库里有两个 Windows 友好的辅助入口：

- [start-here.cmd](../start-here.cmd)：Windows 启动入口。
- [check-codex-env.ps1](../check-codex-env.ps1)：环境检查脚本。

请注意：

- `start-here.cmd` 是启动入口，不是保证成功的一键安装器。
- `check-codex-env.ps1` 是环境检查脚本，不是万能修复工具。
- 企业设备、安全软件、PowerShell 执行策略、网络环境，都可能影响结果。

如果脚本失败，不代表你不适合学 Codex。很多时候只是系统环境需要先排查。
