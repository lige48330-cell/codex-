# Codex 新手起步问题

这份文件不是模板大全。  
它只解决一件事：

**如果你是第一次接触 Codex，第一轮最值得问的到底是哪几句。**

对纯新手来说，真正有用的不是问很多，  
而是先问对。

---

## 先记住一条

第一次不要一上来就问：

- 帮我做个网站
- 帮我做个系统
- 帮我完成一个项目

先从“小问题、小任务、小动作”开始。  
这样你更容易真正感受到 Codex 的价值。

---

## 第一轮最值得问的 5 句

### 1. 先搞懂它是什么

```text
请把我当成完全新手。
不要默认我懂代码，也不要讲太多专业词。

请你用最简单的话告诉我：
1. Codex 是什么
2. 它和普通聊天 AI 有什么不一样
3. 它最适合帮我做什么
```

---

### 2. 先搞懂我第一步该做什么

```text
我现在是完全新手。
请不要一下子讲太多内容。

请你只告诉我：
1. 我现在第一步该做什么
2. 第二步该做什么
3. 哪些内容我现在先不用学
```

---

### 3. 先搞懂怎么尽早用起来

```text
我现在不想先学很多原理。
我更想先知道怎么尽快开始用。

请你用最简单的话告诉我：
1. 我应该先做什么
2. 如果默认方式不好用，还有没有别的办法
3. CC Switch 和 API，我现在需要先理解到什么程度
```

---

### 4. 先拿一个小任务试感觉

```text
请把我当成完全新手。
我想先试一个最简单的小任务，感受一下 Codex 能怎么帮我。

请你给我：
1. 3 个适合新手马上尝试的小任务
2. 每个任务为什么适合新手
3. 我应该先选哪一个
```

---

### 5. 先把一个模糊想法理清

```text
我现在脑子里有一个模糊想法，但我说不太清楚。
请你把我当成完全新手，帮我把这个想法整理清楚。

请你帮我：
1. 先理解我现在大概想做什么
2. 帮我把这个想法整理成更清楚的话
3. 帮我拆成几个简单步骤
4. 告诉我我最适合先做哪一步
```

---

## 如果你只想先复制一句，就用这句

```text
请把我当成完全新手。
不要默认我懂代码，也不要讲太多专业词。

请你：
1. 用最简单的话解释
2. 告诉我第一步该做什么
3. 告诉我哪些内容我现在可以先跳过
4. 告诉我怎样最快开始用起来
```

---

## 最后一个提醒

你第一次接触 Codex，不需要一下子全懂。  
你只需要先做到这几件事：

1. 知道它大概是什么
2. 知道第一步该做什么
3. 知道默认方式不顺时，可以去看 `CC Switch` 和其他 API 的接入方式
4. 先拿一个小任务试出感觉

这样就已经是一个很好的开始了。
